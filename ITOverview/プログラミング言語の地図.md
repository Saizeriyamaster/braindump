# プログラミング言語の地図

> 要点：Python=AI/データ分析、JavaScript=Web画面、TypeScript=大規模開発のJS。「なぜその言語なのか」がわかれば十分

---

## 主要言語の一覧

| 言語 | 主な用途 | 特徴 |
|------|---------|------|
| **Python** | AI/ML、データ分析、スクリプト、バックエンドAPI | 書きやすい。エコシステムが豊富。速くはないがNumPy等で高速化 |
| **JavaScript** | フロントエンド（Web画面） | ブラウザで唯一動く言語。Node.jsでサーバー側にも使える |
| **TypeScript** | フロントエンド〜バックエンド | JavaScriptに型を追加。大規模開発のデファクト |
| **Go** | バックエンドAPI、インフラツール | コンパイル言語で高速。並行処理が得意。Docker, Kubernetes はGo製 |
| **Rust** | システムプログラミング、高性能ツール | 最速・メモリ安全。学習コスト高い。WebAssemblyにも使われる |
| **Java** | 大企業のバックエンド、Android | 歴史が長い。エンタープライズに多い |
| **C / C++** | OS、組み込み、ゲームエンジン、数値計算ライブラリ | 最速だがメモリ管理を手動で行う。PyTorchのコアはC++ |
| **Swift** | iOSアプリ開発 | Apple製。Objective-Cの後継 |
| **Kotlin** | Androidアプリ | Javaの後継。Googleが推奨 |
| **SQL** | データベース操作 | 言語というよりクエリ言語。DB操作の共通言語 |
| **Bash / Shell** | Linuxのスクリプト、CI/CD | サーバー操作・自動化に必須 |
| **R** | 統計分析・可視化 | 統計・バイオインフォマティクス分野 |

## Python をもう少し深掘り

AIエンジニアとして知っておくと会話できる周辺知識：

### なぜPythonがAIに使われるか
- NumPy, PyTorch, TensorFlow などの高性能ライブラリがPython向けに揃っている
- Jupyter Notebook でインタラクティブに試せる
- 書きやすさと豊富なコミュニティ → 研究者が使いやすい

### バージョンと型ヒント
- Python 3.10以降が現在の推奨。f-string（`f"Hello {name}"`）、型ヒントが便利
- 型ヒント（`def predict(text: str) -> float:`）はTypeScriptのように型エラーを静的解析できる
- `pyproject.toml` や `uv` を使った依存関係管理が最新のデファクト

### パッケージ管理
| ツール | 説明 |
|--------|------|
| **pip** | Python標準のパッケージインストーラー |
| **conda** | 仮想環境+パッケージ管理。データサイエンス系で普及 |
| **poetry** | 依存関係の解決が賢い。プロジェクト管理向け |
| **uv** | 最近急速に普及。pipより100倍速い。Rustで書かれている |

## JavaScript / TypeScript の世界

### なぜJSしかフロントで使えないのか
- ブラウザが実行できるのはHTML/CSS/JSのみ（仕様）
- 最近はWebAssemblyでRustやC++をブラウザで動かすことも可能になってきた

### Node.js とは
- Chrome V8エンジンを使ってJSをブラウザ外（サーバー・ターミナル）で動かす仕組み
- npm / yarn / pnpm はNode.jsのパッケージ管理ツール

### TypeScript が使われる理由
- 型がないJSは規模が大きくなると「この変数に何が入るか」わからなくなる
- 型エラーをコンパイル時に検出できる → バグを早期発見
- VSCodeとの相性が良く補完が強力

---

## Go が選ばれる場面

- 高スループットなAPIサーバー、マイクロサービス
- インフラツール（Docker, K8s, Terraform はGo製）
- 並行処理（goroutine）が言語レベルでサポートされている
- Python/Node.jsより10〜100倍速いことも

---

## Rust が注目される理由

- C/C++並みの速度 + メモリ安全（バッファオーバーフロー等がコンパイルで防げる）
- AI/ML推論エンジン、WebAssembly、ブロックチェーン関連で採用増
- PyO3でPythonの拡張モジュールをRustで書ける
- 学習コストは非常に高い

## DSL（ドメイン固有言語）

特定の用途向けに作られたミニ言語：

| DSL | 用途 |
|-----|------|
| **SQL** | データベースクエリ |
| **HTML/CSS** | Webページの構造と装飾 |
| **Regex（正規表現）** | テキストパターンマッチング |
| **GraphQL** | API仕様の定義とクエリ |
| **YAML/JSON Schema** | データ構造の定義 |
| **Dockerfile** | コンテナビルド手順の記述 |

## 言語の選び方（実務の判断軸）

```
AIモデルの学習・推論 → Python（一択）
フロントエンド → TypeScript + React/Next.js
高性能バックエンドAPI → Go or Python(FastAPI)
インフラ自動化スクリプト → Python or Bash
モバイルアプリ → Swift（iOS）or Kotlin（Android）
データ分析 → Python or SQL
```
