# アルゴリズムという考え方

> 要点：「辞書の引き方」で考える。先頭からめくるvs真ん中で半分に割る。効率の良い手順を設計するのがプログラミングの本質

---

## アルゴリズムとは

**アルゴリズム**：問題を解くための「手順」。同じ結果を出すにも、効率の良い手順と悪い手順がある。

例：100万人の名前リストから「田中」を探す
- **線形探索**：先頭から1件ずつ確認 → 最悪100万回
- **二分探索**：リストをソートして真ん中から探す → 最悪20回（log₂(100万) ≈ 20）

---

## 計算量（Big O記法）

アルゴリズムの効率を「データ量nに対してどれだけ処理ステップが増えるか」で表す記法。

| 記法 | 名称 | 意味 | 例 |
|------|------|------|----|
| **O(1)** | 定数時間 | データ量に関係なく一定 | 配列の要素をインデックスで取得 |
| **O(log n)** | 対数時間 | 超効率的。データが2倍でも1ステップ増えるだけ | 二分探索 |
| **O(n)** | 線形時間 | データに比例して増える | 配列の全要素をループ |
| **O(n log n)** | 線形対数 | ソートアルゴリズムの平均的な効率 | マージソート |
| **O(n²)** | 二乗時間 | データが2倍で処理が4倍 | 二重ループ、バブルソート |
| **O(2ⁿ)** | 指数時間 | データが少し増えると爆発的に増加 | 全探索、組み合わせ列挙 |

```
速い ←                          → 遅い
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ)
```

## よく出るアルゴリズム

### 探索

**線形探索（Linear Search）**
- リストを先頭から1件ずつ探す
- O(n)

**二分探索（Binary Search）**
- ソート済みリストの真ん中を見て、前半か後半かに絞り込む
- O(log n)

```python
# 二分探索のイメージ
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

### ソート（並び替え）

| アルゴリズム | 計算量 | 特徴 |
|------------|--------|------|
| **バブルソート** | O(n²) | 隣同士を比較して交換。最も単純だが遅い |
| **マージソート** | O(n log n) | 半分に分けて再帰的にソートして合体 |
+ **クイックソート** | O(n log n)平均 | 実用的に最速。Pythonの`sorted()`内部で使われる |
| **ヒープソート** | O(n log n) | ヒープ構造を使う。最悪ケースも安定 |

実務でソートアルゴリズムを自分で実装することはほぼない（標準ライブラリを使う）が、「なぜソートが O(n log n) なのか」は説明できると良い。

---

### グラフ・木の探索

**BFS（幅優先探索）**
- 近いところから順番に探す
- 最短経路を求めるときに使う

**DFS（深さ優先探索）**
- 1つの経路を深く掘り下げてから戻る
- 迷路の解法、ファイルシステムの走査

## データ構造

アルゴリズムの効率はデータ構造の選択に依存する。

| データ構造 | 特徴 | 主な操作の計算量 |
|-----------|------|----------------|
| **配列（Array）** | 連続したメモリ領域。インデックスアクセスが速い | アクセス: O(1)、検索: O(n) |
| **連結リスト** | ポインタでつながったノード。挿入・削除が速い | アクセス: O(n)、挿入: O(1) |
| **ハッシュテーブル（dict）** | キーから値を即座に取得 | アクセス: O(1)平均 |
| **スタック** | 後入れ先出し（LIFO）。関数呼び出しの管理 | プッシュ/ポップ: O(1) |
| **キュー** | 先入れ先出し（FIFO）。タスクの順番処理 | エンキュー/デキュー: O(1) |
| **二分探索木** | ソート順を保ちながら挿入・検索 | 検索・挿入: O(log n) |
| **ヒープ** | 最大値・最小値を常にO(1)で取得 | 取得: O(1)、挿入: O(log n) |

## 再帰（Recursion）

関数が自分自身を呼び出す手法。分割統治法で使われる。

```python
def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n - 1)  # 自分を呼ぶ
```

- **スタックオーバーフロー**：再帰が深すぎるとメモリが枯渇するエラー
- 深い再帰は**動的計画法（メモ化）**やイテレーションに書き換えることが多い

## AIとアルゴリズムの接点

データサイエンス・AIエンジニアに直接関係する部分：

- **k-NN**：近傍探索アルゴリズム。ベクトルDBの類似検索（ANN: 近似最近傍探索）に使われる
- **ソート**：ランキングモデルの出力整理
- **ハッシュ**：特徴量のキャッシュ、データ重複排除
- **グラフ探索**：知識グラフの推論、GNN（グラフニューラルネットワーク）の前提
