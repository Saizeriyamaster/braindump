# データ形式の基本

> 要点：CSV=シンプルな表、JSON=入れ子にできる構造化データ。APIのやり取りはほぼJSON

---

## CSV（Comma-Separated Values）

行と列のシンプルな表形式。Excelで開ける。

```csv
name,age,role
田中,28,エンジニア
山田,32,デザイナー
```

- 構造がシンプルで人間にも読みやすい
- 入れ子（ネスト）できない → 複雑なデータ構造には不向き
- 機械学習のデータセット、Excel連携でよく使う
- Pandasで読み込む：`pd.read_csv("data.csv")`

---

## JSON（JavaScript Object Notation）

APIのやり取りに使われる主流フォーマット。入れ子構造が書ける。

```json
{
  "name": "田中",
  "age": 28,
  "skills": ["Python", "SQL", "TensorFlow"],
  "address": {
    "city": "東京",
    "zip": "100-0001"
  }
}
```

- ネスト、配列が使える → 複雑なデータ構造を表現できる
- REST APIのレスポンスはほぼJSON
- Pythonでは辞書（`dict`）と対応する
  ```python
  import json
  data = json.loads('{"name": "田中"}')  # 文字列→dict
  json.dumps(data)                        # dict→文字列
  ```

---

## YAML（YAML Ain't Markup Language）

設定ファイルによく使われる。JSONより人間が読みやすい。

```yaml
name: 田中
age: 28
skills:
  - Python
  - SQL
address:
  city: 東京
  zip: 100-0001
```

- GitHub Actions, Docker Compose, Kubernetes の設定ファイルはYAML
- インデント（スペース）が文法の一部なので、スペース数を間違えるとエラーになる
- JSONとの相互変換が可能

---

## XML（eXtensible Markup Language）

タグで囲む形式。HTMLに似た構造。

```xml
<person>
  <name>田中</name>
  <age>28</age>
</person>
```

- 古いエンタープライズシステム、SOAP APIで使われていた
- 今は新規開発ではJSONに置き換わっている
- Excel（.xlsx）の内部形式はXML

---

## Parquet

列指向のバイナリ形式。データ分析・ML向け。

```python
import pandas as pd
df.to_parquet("data.parquet")
df = pd.read_parquet("data.parquet")
```

- CSVの10〜100倍の読み込み速度
- 圧縮効率が高い（ファイルサイズが小さい）
- 特定の列だけを読み込める（CSVは全列読む必要がある）
- BigQuery, Spark, PyArrowで使われる形式
- 機械学習の大規模データ前処理に使う場面が多い

## その他の重要な形式

| 形式 | 用途 |
|------|------|
| **Markdown (.md)** | ドキュメント、READMEの記述形式 |
| **JSONL（JSON Lines）** | 1行1JSONの形式。LLMの学習データ、ストリームログ |
| **Protocol Buffers (Protobuf)** | Googleが開発したバイナリシリアライズ形式。高速・小容量。gRPC通信で使う |
| **Arrow** | インメモリ列指向フォーマット。Pandas, Polars, Sparkの内部で使われる |
| **HDF5** | 大規模配列データの保存。深層学習のモデルウェイト（Keras等） |
| **SQLite (.db)** | ファイル1つのSQL DB。軽量な永続化に |

## エンコーディング

テキストをバイト列に変換する規則。文字化けの原因。

| エンコーディング | 特徴 |
|----------------|------|
| **UTF-8** | 現代のデファクト。日本語含む全言語対応 |
| **Shift-JIS** | 古い日本語エンコーディング。Windowsの日本語ファイルに多い |
| **UTF-16** | WindowsのAPIで使われる。Pythonのstr内部形式 |

Pandasでエラーになるときはだいたい `encoding="utf-8"` か `encoding="shift-jis"` の指定が抜けている。

---

## Base64

バイナリデータをテキストで表現するエンコーディング方式。

- 画像をJSON内に埋め込む、メールに添付ファイルを含める等で使われる
- データ量が約33%増える
- Pythonでは `base64.b64encode()` / `base64.b64decode()`
