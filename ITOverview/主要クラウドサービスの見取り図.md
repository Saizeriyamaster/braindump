# 主要クラウドサービスの見取り図

> 要点：名前と「何に使うか」を紐付ける。プロジェクトで名前が出たとき何のことかわかればOK

---

## カテゴリ別マップ

### コンピューティング（サーバー）

| サービス名 | クラウド | 種別 | 一言説明 |
|-----------|---------|------|---------|
| **EC2** | AWS | IaaS | 仮想サーバー。何でもできる |
| **Compute Engine** | GCP | IaaS | GCPの仮想サーバー |
| **Azure VM** | Azure | IaaS | Azureの仮想サーバー |
| **Lambda** | AWS | サーバーレス | 関数単位で実行、使った分だけ課金 |
| **Cloud Functions** | GCP | サーバーレス | GCPのLambda相当 |
| **Cloud Run** | GCP | コンテナPaaS | コンテナをサーバーレスで動かす。最近人気 |
| **ECS / EKS** | AWS | コンテナ | DockerコンテナをAWSで動かす（ECS:シンプル、EKS:K8s） |

---

### ストレージ

| サービス名 | クラウド | 一言説明 |
|-----------|---------|---------|
| **S3** | AWS | オブジェクトストレージ。ファイル・画像・モデルウェイトの保存 |
| **Cloud Storage (GCS)** | GCP | GCPのS3相当 |
| **EBS** | AWS | EC2にアタッチするブロックストレージ（HDDみたいなもの） |
| **EFS** | AWS | 複数サーバーで共有できるネットワークファイルシステム |

S3/GCSはAIの学習データやモデルウェイトの保管場所として頻出。

---

### データベース

| サービス名 | クラウド | 種別 | 一言説明 |
|-----------|---------|------|---------|
| **RDS** | AWS | RDBMS | PostgreSQL/MySQLをマネージドで使える |
| **Aurora** | AWS | RDBMS | RDSの高性能版。互換性あり |
| **Cloud SQL** | GCP | RDBMS | GCPのRDS相当 |
| **DynamoDB** | AWS | NoSQL(KV) | フルマネージドのNoSQL。大規模向け |
| **Firestore** | GCP | NoSQL | Googleのドキュメント型DB。Firebaseでも使う |
| **BigQuery** | GCP | DWH | ペタバイト規模の分析DB。SQLで大量データ処理 |
| **Redshift** | AWS | DWH | AWSのデータウェアハウス |
| **Snowflake** | マルチ | DWH | クラウドに依存しないDWH。企業で人気 |
| **ElastiCache** | AWS | キャッシュ | Redis/Memcachedをマネージドで |

---

### ネットワーク

| サービス名 | クラウド | 一言説明 |
|-----------|---------|---------|
| **VPC** | AWS/GCP | 仮想プライベートネットワーク。リソースを隔離する |
| **CloudFront** | AWS | CDN（コンテンツ配信網）。静的ファイルを世界中から高速配信 |
| **API Gateway** | AWS | APIの玄関口。認証・レート制限・ルーティングを担う |
| **Load Balancer** | AWS/GCP | 複数サーバーへのアクセスを分散する |
| **Route 53** | AWS | DNSサービス |

---

### AI / ML

| サービス名 | クラウド | 一言説明 |
|-----------|---------|---------|
| **SageMaker** | AWS | ML学習〜推論〜デプロイまでのフルスタックMLプラットフォーム |
| **Vertex AI** | GCP | GCPのSageMaker相当。Google独自モデルも使える |
| **Bedrock** | AWS | Claude, Llama等の基盤モデルをAPIで使えるサービス |
| **Azure OpenAI Service** | Azure | OpenAIモデルをAzure経由で使えるサービス（企業向け） |
| **BigQuery ML** | GCP | SQLでMLモデルを学習・推論できる |

---

### 開発・DevOps

| サービス名 | クラウド | 一言説明 |
|-----------|---------|---------|
| **CloudWatch** | AWS | ログ収集・メトリクス監視・アラート |
| **Cloud Logging / Monitoring** | GCP | GCPの監視サービス |
| **CodePipeline / CodeBuild** | AWS | CI/CDパイプライン |
| **Cloud Build** | GCP | GCPのCI/CDサービス |
| **IAM** | AWS/GCP | Identity and Access Management。誰が何にアクセスできるか管理 |
| **Secrets Manager** | AWS | APIキーやパスワードを安全に保管 |

## Firebase vs Supabase

フルスタックアプリを素早く作るためのBaaS（Backend as a Service）

| | Firebase | Supabase |
|-|----------|---------|
| DB | Firestore（NoSQL） | PostgreSQL（SQL） |
| 認証 | Firebase Auth | Supabase Auth |
| ストレージ | Firebase Storage | Supabase Storage |
| リアルタイム | ネイティブ対応 | 対応あり |
| OSSか | ×（Googleクローズド） | ○（self-hostできる） |
| 選ぶ理由 | Googleエコシステム、リアルタイム重視 | SQLが使いたい、OSSが良い |

## よくある構成パターン

**スタートアップの典型スタック（AWS）**
```
フロント: Vercel (Next.js)
バックエンド: Lambda or ECS (FastAPI/Node.js)
DB: RDS (PostgreSQL)
ファイル: S3
監視: CloudWatch
```

**データ分析基盤（GCP）**
```
データ収集: Cloud Pub/Sub
ストレージ: GCS
処理: BigQuery + dbt
可視化: Looker Studio / Redash
ML: Vertex AI
```
