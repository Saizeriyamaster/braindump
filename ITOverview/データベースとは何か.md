# データベースとは何か

> 要点：Excelの超高性能版。「テーブル」「レコード」「クエリ」の3つだけ覚える

---

## 基本概念

| 用語 | 意味 | Excel対応 |
|------|------|-----------|
| **テーブル** | 同じ種類のデータをまとめた表 | シート |
| **カラム（列）** | 属性の種類（名前、年齢など） | 列 |
| **レコード（行）** | データ1件 | 行 |
| **クエリ** | DBへの問い合わせ命令 | フィルタ・VLOOKUP的な操作 |
| **スキーマ** | テーブルの設計図（どんな列があるか） | — |

---

## RDBMS（リレーショナルDB）

最も一般的なDB。テーブル同士を「関係（リレーション）」でつなぐ。

```sql
-- 例：usersテーブルからageが30以上のユーザーを取得
SELECT name, age FROM users WHERE age >= 30;

-- テーブルを結合（JOIN）する
SELECT users.name, orders.amount
FROM users
JOIN orders ON users.id = orders.user_id;
```

### 主要なRDBMS

| DB | 特徴 | よく使われる場面 |
|----|------|----------------|
| **PostgreSQL** | 高機能、拡張性高い、OSSでデファクト | WebアプリのメインDB |
| **MySQL** | 軽量・高速、歴史が長い | WordPress等の旧来システム |
| **SQLite** | ファイル1つで完結、サーバー不要 | 開発環境、モバイルアプリ |
| **BigQuery** | Google製、超大規模データ分析向け | データウェアハウス |

---

## NoSQL

「テーブル＋SQL」以外のDB。柔軟な構造が必要なときや高速読み書きが必要なときに使う。

| 種類 | 例 | 使い所 |
|------|----|--------|
| **ドキュメント型** | MongoDB, Firestore | ユーザープロフィール、JSONデータ |
| **KVS（キーバリュー）** | Redis, DynamoDB | キャッシュ、セッション管理 |
| **カラム型** | Cassandra, BigTable | 大量書き込み、時系列データ |
| **グラフ型** | Neo4j | 友達関係、知識グラフ |
| **ベクトルDB** | Pinecone, Weaviate, pgvector | 類似検索、RAG | ← AIエンジニアに一番関係する

---

## RDBMSの重要概念

### 主キー（Primary Key）と外部キー（Foreign Key）
- **主キー**：テーブル内でレコードを一意に識別するカラム（例：`user_id`）
- **外部キー**：別テーブルの主キーを参照するカラム。テーブル間の関係を表す

```
users テーブル: id, name, email
orders テーブル: id, user_id(外部キー), amount, created_at
```

### トランザクション（ACID特性）
複数の操作を「全部成功か全部失敗か」として扱う仕組み。

- 例：銀行振込で「A口座から引き落とし」と「B口座に入金」は必ずセットで成功する必要がある
- **ACID**：Atomicity（原子性）、Consistency（一貫性）、Isolation（分離性）、Durability（永続性）

### インデックス
- 特定のカラムに「索引」をつけて検索を高速化する仕組み
- 本の索引と同じ。インデックスなしで全行スキャンは本を全ページ読む状態
- 多すぎると書き込みが遅くなるトレードオフあり

## Supabase / Firebase（BaaS）

バックエンドをゼロから作らずに使えるサービス。

| | Supabase | Firebase |
|-|----------|---------|
| 基盤 | PostgreSQL（SQL使える） | Firestore（NoSQL） |
| 提供元 | OSSプロジェクト | Google |
| 認証 | あり | あり |
| 使い所 | PostgreSQLが欲しい × 手軽に | リアルタイムDB × Googleエコシステム |

---

## データウェアハウス（DWH）vs データレイク

| | DWH | データレイク |
|-|-----|------------|
| データ形式 | 構造化（テーブル） | 何でもあり（CSV, JSON, 画像等） |
| 目的 | BIレポート・集計分析 | 探索・AI学習データ |
| 例 | BigQuery, Redshift, Snowflake | S3, GCS, Azure Data Lake |
