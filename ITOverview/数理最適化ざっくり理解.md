# 数理最適化ざっくり理解

> 要点：配送ルート、シフト表、広告予算配分など「制約の中でベストを選ぶ」問題。AIとは別の技術だが案件で出てくる

---

## 数理最適化とは

**目的関数を最大化（または最小化）しながら、制約条件を満たす変数の値を求める技術**

```
例：広告予算配分
　目的：ROI（投資収益率）を最大化したい
　変数：各チャンネル（TV/Web/SNS）への予算配分額
　制約：総予算は1億円以内、TV予算は3000万円以上
```

機械学習（パラメータの最適化）とは違い、**明示的な制約条件**があるのが特徴。

---

## 種類の整理

### 線形計画法（LP: Linear Programming）

目的関数も制約条件も「線形（一次式）」の場合。

```
最大化: 3x + 5y
制約:
  x + y ≤ 4
  x ≤ 3
  y ≤ 3
  x, y ≥ 0
```

- シンプレックス法などで効率よく解ける
- 工場の生産計画、物流コスト最適化に使われる

### 整数計画法（IP: Integer Programming）

変数が整数（0か1など）に限定される。

- **0-1整数計画法**：変数が0か1 = 「やる/やらない」の選択
- ナップサック問題、シフト最適化、施設配置
- 連続変数の最適化より計算コストが高い（NP困難なものも多い）

### 混合整数計画法（MIP: Mixed Integer Programming）

連続変数と整数変数が混在。現実問題のほとんどはこれ。

### 非線形計画法（NLP: Nonlinear Programming）

目的関数や制約が非線形の場合。ディープラーニングの学習もこの一種（ただし手法が異なる）。

## 代表的な問題

### 巡回セールスマン問題（TSP）

n都市を全て一度ずつ訪問して出発点に戻る最短ルートを求める。

- 都市が20でも解の候補は19! ≈ 10^17 通り
- NP困難（多項式時間で厳密解が求められない）
- 実務では**ヒューリスティック法**や**近似解法**を使う

### ナップサック問題

容量制限のあるナップサックに価値と重さの異なるアイテムを詰める。価値を最大化せよ。

- 広告予算配分、ポートフォリオ最適化に類似した問題

### シフト最適化

- スタッフの希望・スキル・必要人数・労働規制を制約に、コストや満足度を最適化
- 医療・小売・コールセンターで使われる

### 配送ルート最適化（VRP）

複数の配達先を効率よく回る複数車両のルートを決める。Amazonの物流にも使われている。

## AIとの違い・使い分け

| | 数理最適化 | 機械学習・AI |
|-|-----------|------------|
| アプローチ | 数式でモデル化して解く | データからパターンを学習 |
| 制約条件 | 明示的に設定できる | 学習に組み込むのが難しい |
| 解釈可能性 | 高い（なぜその解か説明できる） | 低いことが多い |
| データ不要 | 不要（数式モデルで解く） | 大量データが必要 |
| 使い所 | 制約が明確な意思決定問題 | 予測・分類・生成 |

実務では**ML（予測）+ 最適化（意思決定）**の組み合わせがよくある。
例：需要予測（ML） → 在庫・発注最適化（MIP）

## ソルバー（解くツール）

| ソルバー | 特徴 |
|---------|------|
| **PuLP** | PythonのOSSライブラリ。入門向け |
| **OR-Tools** | Google製。組み合わせ最適化に強い |
| **Gurobi** | 商用の高性能ソルバー。産業用途のデファクト |
| **CPLEX** | IBM製の商用ソルバー |
| **GLPK** | 無料OSSソルバー |

```python
from pulp import *

# 線形計画問題を解く例
prob = LpProblem("sample", LpMaximize)
x = LpVariable("x", lowBound=0)
y = LpVariable("y", lowBound=0)

prob += 3*x + 5*y     # 目的関数
prob += x + y <= 4    # 制約1
prob += x <= 3        # 制約2
prob += y <= 3        # 制約3

prob.solve()
print(x.value(), y.value())  # → 1.0, 3.0
```

## メタヒューリスティクス（近似解法）

厳密解が求めにくい問題には近似解法を使う：

| 手法 | イメージ |
|------|---------|
| **遺伝的アルゴリズム（GA）** | 生物の進化をシミュレート。交叉・突然変異で解を改善 |
| **焼きなまし法（SA）** | 金属の焼きなまし過程に着想。確率的に悪い解も受け入れる |
| **粒子群最適化（PSO）** | 鳥の群れの行動をシミュレート |
| **強化学習（RL）** | 報酬を最大化する行動方策を学習。AlphaGoもこれ |
